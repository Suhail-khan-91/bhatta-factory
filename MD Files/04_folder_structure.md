# Phase 4: Next.js App Router Folder Structure

## Core Architecture Rule
* **Pages (`app/`):** Act strictly as routing shells. They contain layout and import components.
* **Components (`components/`):** Hold 100% of the business logic, UI state, and database fetching.

## 1. The Routing Directory (`/app`)
This defines the actual URLs of the website (e.g., `yoursite.com/master-entry`).

```text
/app
  layout.js                 # Global wrapper (Contains the BottomNavBar so it shows on every page)
  page.js                   # The Root URL (Auto-redirects users to /master-entry after login)
  /calculator
    page.js                 # Tab 1: The Calculator Shell
  /master-entry
    page.js                 # Tab 2: The Master Entry Shell (Displays the square buttons)
  /analytics
    page.js                 # Tab 3: Stats & Analytics Shell
  /payroll
    page.js                 # Tab 4: Salary & Payroll Shell
  /sales
    page.js                 # Tab 5: Brick Sales Shell




## 2. The Components Directory (/components) 
This is where the actual feature files live. When the AI builds a feature, it creates a file here.

Plaintext
/components
  /navigation
    BottomNavBar.js         # The fixed 5-button Instagram-style navigation bar
  
  /calculator
    CalculatorWidget.js     # Client-side math logic and history UI
  
  /master-entry
    FuelInputForm.js        # Form for buying fuel (Uploads to fuel_slots table)
    FuelOutputForm.js       # Form for using fuel (Uploads to fuel_usage_logs table)
    TeamProductionModal.js  # Popup to enter daily brick counts
    EmployeeOnboarding.js   # Form to add new workers and upload Aadhar photos
  
  /analytics
    FuelProgressBar.js      # Visual line showing current fuel slot remaining
    FuelHistoryList.js      # Accordion UI for past fuel slots
    ProductionChart.js      # Graph showing bricks made over time
  
  /ui                       # (Auto-generated by shadcn/ui)
    button.js
    input.js
    dropdown.js
